# 🎨 宠物AI App · UI设计需求文档 v2.0

> 文档日期：2026年3月 · 版本：v2.0 · 适用范围：iOS App（Expo EAS构建）· 目标市场：韩国 + 中国
>
> v2.0 更新：重构导航架构，补充问诊/档案/心理评估/翻译页面规范，新增 Emergency Alert 组件，色彩系统补充警告红。

---

## 一、设计哲学

本产品的核心情感承诺是「安全感」，而不是「诊断工具」。UI设计必须在三个维度上支撑这一承诺：

**专业可信**：用户要相信这个App给出的建议是科学的，不是随意的。

**有温度**：宠物生病时，用户处于焦虑状态。界面必须像一个懂事的朋友，而不是冷冰冰的医疗系统。

**渐进披露**：不要一次性把所有信息砸给用户。先给结论，再给细节，让用户自己决定要了解多深。

**克制简洁**：每个页面只做一件事。功能多不等于界面复杂。导航清晰、操作路径短是韩国市场用户的核心期待。

## 二、导航架构（信息架构 IA）

### 2.1 底部 Tab Bar（4个固定入口）

```
┌──────────┬──────────┬──────────┬──────────┐
│  🏠 홈   │  🔍 검사 │  💬 문진 │  📋 기록 │
│  首页    │  检测    │  问诊    │  档案    │
└──────────┴──────────┴──────────┴──────────┘
```

5个功能模块，4个Tab视觉：

| Tab | 核心职责 | 不包含 |
|-----|---------|-------|
| 首页 | 今日状态总览 + 快捷入口 | 不做检测，不做问诊 |
| 检测 | 10个视觉/声音检测模块 | 不包含历史记录 |
| 问诊 | 多轮AI对话，讨论宠物健康问题 | 不做图像检测，不包含翻译 |
| 档案 | 历史记录 + 心理评估 + 宠物信息 | 不做实时交互 |

翻译模块是**独立页面**，但不占独立常驻Tab位，通过首页快捷入口进入。

> **设计原则**：心理健康评估不占独立Tab，由AI在档案页主动发起，频率低（月度），不需要常驻入口。

### 2.2 页面层级

```
App
├── Tab: 首页
│   ├── 紧急警告弹窗（Emergency Alert）
│   ├── 快捷入口 → 检测各模块
│   └── 快捷入口 → 翻译页
├── Tab: 检测
│   ├── 模块选择页
│   ├── 拍摄引导页（各模块专属）
│   └── 检测结果页
├── Tab: 问诊
│   ├── 对话主界面（多轮AI健康问诊）
│   └── 对话历史列表
├── 独立页：双向翻译（번역）
│   ├── 人 → 宠物模式
│   └── 宠物 → 人模式（调用M9/M10）
└── Tab: 档案
    ├── 宠物信息卡
    ├── 健康时间轴
    ├── 心理健康评估（月度，AI发起）
    └── 设置 / 我的
```

## 三、色彩系统

### 3.1 全局调色板

所有颜色统一在低饱和度「粉彩/马卡龙」色系，整体视觉像加了一层柔光滤镜。

| 用途 | 色值 | 说明 |
|------|------|------|
| 主品牌色 (Brand) | `#BDE0FE` | 清透婴儿蓝，轻盈、洁净、医疗感 |
| 背景底色 (Background) | `#FAFCFF` | 极微弱的蓝白色，营造蓝天白云感 |
| 卡片背景 | `#FFFFFF` | 纯白加玻璃拟态效果 |
| 深色文字 | `#2B3A55` | 深海蓝，与品牌色对比鲜明 |
| 次级文字 | `#7A8DA3` | 中灰蓝，用于说明性文字 |
| 分割线 | `#E8EFF6` | 极浅，不抢眼 |
| 🚨 紧急警告红 (Emergency) | `#D32F2F` | **仅限**本地规则触发的紧急情况，禁止在其他场景使用 |

### 3.2 情绪温度计色系（健康状态专用）

这是贯穿全App最重要的视觉语言，任何表示健康状态的地方都必须使用此色系。

| 状态 | 色值 | 使用场景 |
|------|------|---------|
| 🔵 正常 | `#90CAF9` | 检测结果正常、健康档案无异常 |
| 🟡 注意 | `#FFE29F` | 有轻微异常、建议观察 |
| 🟠 建议就医 | `#FFB5A7` | 检测到明显问题、建议去医院 |
| 🔴 紧急 | `#D32F2F` | 本地规则直接触发，不经VLM |

> **色系语义边界**：🟠 蜜桃橘 = 「建议去医院」；🔴 警告红 = 「现在就去」。橙色有意选用蜜桃橘而非纯红，警示但不引发过度恐慌，符合「陪伴式」而非「恐吓式」定位。警告红是唯一例外——当本地规则检测到真实生命安全风险时，打破视觉一致性。

## 四、字体排版系统

三种字体分别对应三种信息层级，不可混用。

| 字体 | 用途 | 情感作用 |
|------|------|---------|
| **Noto Serif KR Bold** | 页面标题、检测结论关键词 | 衬线体带学术感权威感 |
| **Pretendard Regular/Medium/SemiBold** | 所有正文、按钮、说明文字 | 韩国主流选择，用户熟悉 |
| **Roboto Mono** | 所有数值：体重/日期/检测数值 | 等宽传达精准和科学 |

**字号规范**：主标题 28sp，次级标题 20sp，卡片标题 18sp，正文 16sp，次级说明 14sp，标签 12sp。

## 五、布局系统

### 5.1 核心网格

- 12列网格，屏幕两侧留白 `px-6`（24px）至 `px-8`（32px），留白比例约15%
- 卡片间距：`gap-4`（16px）基准，重要卡片间 `gap-6`（24px）

### 5.2 玻璃拟态（仅限3种场景）

```javascript
// 标准玻璃卡片（expo-blur）
<BlurView intensity={60} tint="light"
  style={{
    borderRadius: 20,
    borderWidth: 0.5,
    borderColor: 'rgba(255,255,255,0.6)',
    backgroundColor: 'rgba(255,255,255,0.4)'
  }}
/>
```

**允许使用玻璃拟态的场景**：① 底部 Tab Bar　② 悬浮情绪温度计卡片　③ 普通 Modal/Alert 弹窗

**禁止使用**：紧急警告弹窗（实色，故意打破轻柔感）、普通信息卡片（纯白 + 品牌蓝弥散阴影）

## 六、交互动效规范

```javascript
// 标准动画（所有页面统一）
{ duration: 300, easing: Easing.bezier(0.25, 0.46, 0.45, 0.94) }

// 按钮点击
{ scale: 0.95, duration: 100 }

// 卡片展开（弹性）
{ damping: 15, stiffness: 150, mass: 1 }

// 紧急警告入场（更重）
{ damping: 20, stiffness: 200, duration: 400 }
```

## 七、核心页面规范

### 7.1 首页（홈 / 首页）

**设计原则**：信息密度低，一眼看懂。不是功能入口的堆砌，是今日状态的摘要。

```
┌─────────────────────────────────┐
│  状态栏                          │
├─────────────────────────────────┤
│  「안녕, 橘子 오늘은 어때?」       │  Noto Serif 24sp
│  上次检测：6小时前                │  Pretendard 14sp 次级色
├─────────────────────────────────┤
│  ┌── 情绪温度计（玻璃拟态）──────┐ │
│  │  🔵 전체적으로 건강해요        │ │  Noto Serif 20sp
│  │  「你是一个细心的宠物家长 ✨」 │ │  Pretendard 14sp 品牌蓝
│  └──────────────────────────────┘ │
├─────────────────────────────────┤
│  快捷检测（2列，4个常用模块）       │  ← 用户可自定义排序
│  ┌──────────┐  ┌──────────┐      │
│  │ 🦷 구강   │  │ 🐾 피부   │      │
│  └──────────┘  └──────────┘      │
│  ┌──────────┐  ┌──────────┐      │
│  │ 👁 눈     │  │ 💩 배변   │      │
│  └──────────┘  └──────────┘      │
├─────────────────────────────────┤
│  AI 今日提示（1条，Pretendard）    │  ← 基于档案数据主动生成
├─────────────────────────────────┤
│  底部 Tab Bar（玻璃拟态）          │
└─────────────────────────────────┘
```

**首页不包含**：完整检测历史（在档案Tab）、对话入口（在问诊Tab）

### 7.2 检测页（검사 / 检测）

**模块选择页**（进入Tab后的第一屏）：

```
┌─────────────────────────────────┐
│  건강 검사                        │  Noto Serif 22sp
│  어디를 확인할까요?               │  Pretendard 14sp 次级
├─────────────────────────────────┤
│  [체형 건강]  ← 分组标签，Pretendard 12sp
│  ┌──────┐ ┌──────┐ ┌──────┐    │  3列网格
│  │ 구강  │ │ 피부  │ │ 귀   │    │
│  └──────┘ └──────┘ └──────┘    │
│  ┌──────┐ ┌──────┐ ┌──────┐    │
│  │ 눈   │ │ 배변  │ │ 구토물│    │
│  └──────┘ └──────┘ └──────┘    │
│                                  │
│  [재미 분석]  ← 第二分组           │
│  ┌──────┐ ┌──────┐ ┌──────┐    │
│  │ 품종  │ │ 감정  │ │ 소리  │    │
│  └──────┘ └──────┘ └──────┘    │
│  ┌──────┐                        │
│  │ 행동  │                        │
│  └──────┘                        │
└─────────────────────────────────┘
```

**拍摄引导页**（进入具体模块后）：

- 取景框：各模块专属形状（眼睛模块=双眼横向椭圆框；口腔=宽矩形；排泄物=正方形+白纸提示）
- 实时质量反馈：亮度不足→提示「光线太暗，靠近光源」；模糊→「靠近一点」
- 排泄物/呕吐物模块：全屏心理破冰文案 + 白纸背景引导

**检测结果页**（渐进式三层）：

- 第一屏：情绪温度计大卡片 + 结论（Noto Serif）+ 情感文案
- 第二屏（展开）：Grad-CAM / 标注叠加图 + 量化数据（Roboto Mono）+ 说明文字
- 第三屏（底部）：「기록 저장」主按钮 + 「근처 병원 예약」次级按钮（仅橙色状态显示）

### 7.3 问诊页（문진 / 问诊）

**设计原则**：对话界面克制极简。问诊只做健康问诊，翻译是独立页面，不在此出现。

```
┌─────────────────────────────────┐
│  ← 宠物头像  橘子의 문진          │  Pretendard SemiBold 18sp
│                    🕐 기록 보기  │  ← 右上角：查看历史对话
├─────────────────────────────────┤
│                                  │
│  ┌──────────────────────────┐   │
│  │ 🤖 AI 气泡（左对齐）        │   │  Pretendard 16sp
│  │ 오늘 橘子 상태 어떤가요?   │   │
│  └──────────────────────────┘   │
│                                  │
│         ┌──────────────────────┐ │
│         │ 用户气泡（右对齐）     │ │  品牌蓝背景 #BDE0FE
│         │ 밥을 안 먹고 있어요   │ │
│         └──────────────────────┘ │
│                                  │
│  ┌──────────────────────────┐   │  ← 快捷回复建议（可关闭）
│  │ 언제부터요? │ 얼마나?    │   │
│  └──────────────────────────┘   │
├─────────────────────────────────┤
│  ┌──────────────────────┐  [↑]  │  ← 输入框 + 发送
│  │ 메시지 입력...        │      │
│  └──────────────────────┘      │
│  [📷 사진 첨부]  [🎤 음성]       │  ← 附件入口（图片直通检测）
└─────────────────────────────────┘
```

**关键设计决策**：

- 问诊 = 用户与AI多轮对话，讨论宠物健康问题，AI有长期记忆（基于档案数据）
- 翻译 = 完全独立的页面（见 7.4），不是问诊的开关或子功能

**图片附件交互**：用户在问诊中上传图片 → 弹出选项「直接发给AI分析」or「进入检测模块精细分析」

### 7.4 双向翻译页（번역 / 翻译）

**产品定位**：这不是语言翻译工具，而是**跨物种沟通桥梁**。两个方向含义完全不同：

- **人→宠物**：把人类语言/情绪转化为宠物可能理解的信号（声音频率、行为指令、肢体语言建议）
- **宠物→人**：解读宠物的声音/行为/肢体语言，翻译成人类可理解的情绪状态描述

> 这是趣味性 + 情感连接的核心模块，不是严肃的医疗功能。语气设计应轻松，结果要有趣味性但保持科学依据（结合M8情绪识别 + M9声音VA模型）。

**页面布局**：

```
┌─────────────────────────────────┐
│  번역                             │  Noto Serif 22sp
│  橘子와 대화해볼까요?             │  Pretendard 14sp 次级
├─────────────────────────────────┤
│  方向切换（Toggle，居中）          │
│  ┌──────────┐    ┌──────────┐   │
│  │ 🧑→🐱    │    │ 🐱→🧑    │   │  两个Tab，300ms过渡
│  │ 나→고양이 │    │ 고양이→나│   │
│  └──────────┘    └──────────┘   │
├─────────────────────────────────┤
│  [人→宠物 模式]                   │
│                                  │
│  「想对橘子说什么？」               │  Pretendard 16sp
│  ┌──────────────────────────┐   │
│  │ 오늘 많이 기다렸지?       │   │  文字输入
│  └──────────┬───────────────┘   │
│             ↓ 翻译               │
│  ┌──────────────────────────┐   │  结果卡片，品牌蓝背景
│  │ 🐱 橘子가 이해할 수 있는  │   │
│  │    신호로 변환했어요       │   │
│  │                           │   │
│  │  • 낮고 부드러운 목소리로  │   │  具体行为建议
│  │    천천히 말해주세요       │   │
│  │  • 눈을 천천히 깜빡이면   │   │
│  │    애정 표현이 돼요 😊     │   │
│  └──────────────────────────┘   │
├─────────────────────────────────┤
│  [宠物→人 模式]                   │
│                                  │
│  输入方式（二选一）                │
│  ┌──────────┐  ┌──────────┐     │
│  │ 🎤 소리   │  │ 📷 행동  │     │  录音 or 拍摄行为
│  │ 녹음하기  │  │ 촬영하기  │     │
│  └──────────┘  └──────────┘     │
│             ↓ 分析（调用M9/M8）  │
│  ┌──────────────────────────┐   │
│  │ 🐱 橘子의 말을 번역했어요 │   │  结果卡片
│  │                           │   │
│  │ 「배고파요! 빨리 밥 주세요│   │  Noto Serif，趣味语气
│  │   라고 하는 것 같아요 🍽️」 │   │
│  │                           │   │
│  │ VA분석: 흥분+긍정 (V+A+)  │   │  Roboto Mono，折叠展示
│  └──────────────────────────┘   │
└─────────────────────────────────┘
```

**技术联动**：

- 宠物→人（声音）：直接调用 M9 Sound VA模型（500ms，无VLM）
- 宠物→人（行为）：调用 M10 行为识别管线
- 人→宠物：纯VLM推理，结合品种知识库生成行为建议

**语气设计原则**：

- 翻译结果必须用「~하는 것 같아요」「~라고 하는 것 같아요」等不确定性表达
- 加入emoji增加趣味性，但不超过2个/条
- 「人→宠物」结果落地到具体可操作的行为建议，不是空洞描述

**Tab Bar位置**：翻译是独立的第5个功能，但Tab Bar维持4个（首页/检测/问诊/档案）。翻译入口从首页快捷入口进入，不占常驻Tab位置，保持界面简洁。

> **设计理由**：翻译功能使用频率高但不是核心健康功能。放在常驻Tab会稀释「健康助手」的产品定位。

### 7.5 档案页（기록 / 档案）

**设计原则**：这是长期陪伴感的核心载体。不是冷冰冰的数据库，是宠物生命历程的可视化。

```
┌─────────────────────────────────┐
│  宠物信息卡（头像 + 基本信息）     │
│  ┌──────────────────────────┐   │
│  │ 🐱  橘子                  │   │  Noto Serif 24sp
│  │      英短 · 3세 · 4.5kg   │   │  Pretendard 14sp + Roboto Mono
│  └──────────────────────────┘   │
├─────────────────────────────────┤
│  心理健康状态卡（AI主动发起）       │
│  ┌──────────────────────────┐   │
│  │ 🟡 이달의 마음 건강        │   │  每月AI评估结果
│  │ 「橘子 이번 달은 약간      │   │  Pretendard 16sp
│  │   불안해 보여요」           │   │
│  │  [자세히 보기 →]           │   │  → 展开评估详情
│  └──────────────────────────┘   │
├─────────────────────────────────┤
│  健康时间轴                       │
│  2026.03.11  口腔检测  🔵 정상   │
│  2026.03.09  皮肤检测  🟡 주의   │
│  2026.03.07  问诊记录  💬        │
│  2026.03.01  品种识别  🐱        │
│              [더 보기]            │
├─────────────────────────────────┤
│  健康趋势图（折线，Victory Native）│
│  颜色使用情绪温度计三色            │
└─────────────────────────────────┘
```

**心理健康评估模块**（非独立Tab，嵌在档案页）：

- 触发方式：AI基于过去30天的检测数据 + 对话记录主动评估，月度推送
- 不是问卷填写，是AI主动出具的分析卡片
- 用户可点击查看依据（「过去30天内，橘子有14次低V高A声音信号」）
- 评估结果参考情绪温度计三色，不引入新状态颜色

## 八、紧急警告组件规范（Emergency Alert）

> 由后端本地规则触发，**不等待VLM返回**，直接覆盖当前屏幕。触发条件：排泄物 `blood_ratio > 5%`、呕吐物血液检测或异形固体 `circularity < 0.3`。

### 触发条件对照表

| 触发场景 | 本地规则 | 颜色 | 优先级 |
|---------|---------|------|-------|
| 排泄物含血 | `blood_ratio > 0.05` | `#D32F2F` | 最高，跳过VLM |
| 呕吐物含血 | `blood_ratio > 0.03` | `#D32F2F` | 最高，跳过VLM |
| 疑似误食异物 | `circularity < 0.3 AND area > threshold` | `#D32F2F` | 最高，跳过VLM |
| 建议就医（普通） | VLM判断 | `#FFB5A7` | 正常流程 |

## 九、按钮设计系统

### 9.1 主按钮（Primary CTA）

```javascript
  color: '#2B3A55',              // 深海蓝，禁止用白色
  shadowColor: '#BDE0FE',
  shadowOffset: { width: 0, height: 6 },
  shadowOpacity: 0.4, shadowRadius: 16,
}
```

### 9.2 次级按钮（Secondary）

```javascript
{
  height: 48, borderRadius: 24,
  backgroundColor: 'rgba(189,224,254,0.4)',
  borderWidth: 0.5,
  borderColor: 'rgba(255,255,255,0.6)',  // 果冻感边框
  color: '#2B3A55',
}
```

### 9.3 紧急按钮（Emergency Only）

```javascript
{
  width: '100%', height: 56,
  borderRadius: 16,              // 比普通按钮方正，传达严肃
  backgroundColor: '#D32F2F',
  color: '#FFFFFF',              // 唯一允许白色文字的按钮
  shadowColor: '#D32F2F',
  shadowOffset: { width: 0, height: 6 },
  shadowOpacity: 0.35, shadowRadius: 14,
}
```

> **为何紧急按钮用白色文字**：红色背景上深蓝文字对比度约2.8:1，不达WCAG AA标准。白色文字对比度5.1:1，达标。这是全局「禁用白色文字」规则的唯一例外，有明确技术依据。

## 十、Grad-CAM 与标注叠加图

热力图让用户看到「AI在看哪里」，是信任建立的核心组件。各模块标注层不同：

| 模块 | 标注叠加图类型 |
|------|--------------|
| 口腔 | 牙龈/牙齿分区色块（两色区分）+ 描边，非热力图 |
| 皮肤 | 分割Mask描边 + Grad-CAM热力图（双层叠加） |
| 眼睛 | 三区域分割（巩膜/虹膜/眼睑）+ 左右对比框 |
| 耳道/排泄物/呕吐物 | 标准Grad-CAM热力图 |

**通用参数**：热力图叠加透明度50%，绿-黄-红渐变（独立于情绪温度计色系），用户可切换「원본/열지도」（300ms渐变）。

## 十一、文案规范

### 11.1 核心情感文案（检测结果必须出现）

| 状态 | 标题文案（韩文） | 情感文案 |
|------|---------------|---------|
| 🔵 正常 | 오늘 상태 양호해요 | 你是一个细心的宠物家长 ✨ |
| 🟡 注意 | 조금 신경 써주세요 | 发现得早，处理起来会轻松很多 |
| 🟠 就医 | 가까운 병원에 가보세요 | 你及时发现了，这非常重要 ❤️ |
| 🔴 紧急 | — | **不使用情感文案**，直接呈现紧急弹窗，精简、明确。 |

### 11.2 语气原则

- 不用医学术语，不说确定性诊断
- 情绪/翻译模块：一律用「~같아요」「~보여요」「~라고 하는 것 같아요」等不确定性表达
- 翻译结果加入emoji增加趣味性，但不超过2个/条
